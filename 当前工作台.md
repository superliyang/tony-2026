## 当前最活跃的两条主线

### 0. Big-Data：数据系统主干启动

- 当前目标：把大数据从工具名列表收成“业务问题 -> 数据生命周期 -> 计算模式 -> 存储形态 -> 治理与成本”的可恢复学习主干
- 当前状态：已经建立 `Big-Data` 领域骨架、第二层主干、核心系统卡，并补上 `支付实时风控`、`游戏实时运营`、`AI 数据供给` 三条端到端案例
- 下一步：转入 `数仓建模 / 指标口径 / semantic layer` 层，让案例中的指标和数据产品更可治理
- 入口：[[01-Areas/Big-Data/专题总览|Big-Data 专题总览]]
- 恢复：[[01-Areas/Big-Data/恢复笔记|Big-Data 恢复笔记]]

### 1. AI-Open-Source：Agent 系统核心 8 样本

- 当前目标：把 `LangGraph / AutoGen / OpenHands / browser-use / MCP Servers / A2A / LiteLLM / Langfuse` 收成一条可教学、可比较、可迁移的核心样本线
- 当前状态：`LiteLLM`、`Langfuse`、`AutoGen`、`browser-use`、`MCP Servers`、`A2A` 已经进入深度项目卡层级
- 下一步：补这 8 个样本的 `组织页 / 模式页 / adoption case / 源码切入点`
- 入口：[[01-Areas/AI-Open-Source/专题总览|AI-Open-Source 专题总览]]
- 恢复：[[01-Areas/AI-Open-Source/恢复笔记|AI-Open-Source 恢复笔记]]

### 2. AI-Engineering：Memory Lab 与自改进插件

- 当前目标：把 `self-improving-learning-ledger + self-improving-memory-lab` 从概念验证推进到可解释、可回放、可验收的本地 lab
- 当前状态：`hook-shaped capture -> ledger -> shadow review -> consolidation -> manual verify -> candidate skill -> incident replay` 这条链已经通了
- 还没最终证实：真实 live Codex hook 是否会稳定原生触发并写出 debug snapshot
- 下一步：继续做 live hook 观察，同时把验收经验整理成更稳定的教学路径
- 入口：[[01-Areas/AI-Engineering/06-Projects/Memory Lab/项目总览|Memory Lab 项目总览]]
- 验收：[[01-Areas/AI-Engineering/06-Projects/Memory Lab/Self-Improving Memory Lab 验收与当前边界|Self-Improving Memory Lab 验收与当前边界]]
