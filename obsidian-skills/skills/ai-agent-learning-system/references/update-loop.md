# Update Loop

Use this maintenance rhythm for the AI agent learning system.

## When adding a new system

- update `Systems Index.md`
- update at least one relevant map
- decide whether a comparison note should be expanded
- decide whether the system reveals a new engineering abstraction

## When adding a new engineering note

- update `AI-Engineering/07-Topics/Topics Index.md`
- update `AI-Engineering/08-Maps/Maps Index.md` when topology changed
- check whether related system pages should link back to the abstraction

## When adding a new application note

- update `AI-Applications/05-Topics/Topics Index.md`
- update industries, case studies, or templates if the new note changes the structure
- refresh progress and resume notes if the recommended reading path changed

## Before checkpointing

- validate wikilinks for touched areas
- keep `.obsidian` out of routine checkpoints
- prefer one meaningful commit over many tiny topology commits
