# Obsidian Topology Patterns

## Minimal Domain Pattern

Use for a small or early-stage domain:

- `README.md`
- `Topics Index.md`
- `Learning Progress.md`
- `Resume Note.md`
- a few topic pages

## Entity-Rich Pattern

Use when the domain has many concrete actors or products:

- companies / organizations
- people
- systems / products
- topics
- maps

This works well for AI ecosystems, software landscapes, and market studies.

## Practice-Oriented Pattern

Use when the domain includes operations or applied workflows:

- topics
- systems or tools
- maps
- templates
- playbooks or runbooks

This works well for engineering, payments, operations, and GTM systems.

## Long-Term Learning Pattern

Use when the user expects to pause and resume often:

- domain README
- topic index
- progress note
- resume note
- map pages
- simple status model such as `seed`, `learning`, `stable`
