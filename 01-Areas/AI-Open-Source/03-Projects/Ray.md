# Ray

## 它是什么

分布式 runtime 与 AI compute engine，连接训练、推理与数据处理工作负载。

## 为什么值得关注

它很适合研究“分布式 runtime”这条开源路线，尤其是和 serving / agent / orchestration 之间的边界。

## 核心抽象

- distributed runtime
- AI compute engine
- Ray Serve / KubeRay 生态

## 当前建议的学习动作

- 先判断它到底是底座、壳层、平台还是子系统
- 看它和相邻项目是替代关系还是组合关系
- 再决定是本地实验、架构学习，还是生产研究
