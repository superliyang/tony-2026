# GitHub 开源项目 Intake 工作流

1. 先判断项目属于哪一类：runtime、serving、agent、memory、eval、platform
2. 再看背后组织：这是一条稳定路线，还是一次性仓库
3. 建项目卡：不抄 README，要回答“为什么值得学”
4. 把它接到模式页，而不是只留在项目页
5. 判断是否进入 watchlist
6. 如果研究方法有新的稳定偏好，交给 `self-improving-memory-lab` 旁路沉淀
