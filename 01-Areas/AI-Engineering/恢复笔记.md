# 恢复笔记

## Last Session

- We pushed the MLOps / LLMOps branch from lifecycle basics into enterprise platform tradeoffs, then expanded the AI security branch into a more usable control stack with practical cases, release gates, reusable templates, and an explicit enterprise operating model.

## What I Actually Understand Now

- Enterprise `LLMOps` choice is really about choosing a control point and a deployment model, not about chasing a single all-in-one product.
- `MLflow`, `W&B`, `Langfuse`, `Phoenix`, and `Promptfoo` sit on different layers of the lifecycle.
- AI security must cover input, context, tools, state, and artifacts, not just model outputs.
- Prompt injection defense is inseparable from tool safety and approval design.
- Model supply-chain security is now a real engineering concern, not an edge case.

## What Still Feels Fuzzy

- We now also have a Mac-first execution lane, but it still needs real hands-on runs to become skill.
- AI security now has much better case, release-gate, and operating-model language, but still deserves deeper enterprise incident coverage later.
- The new security templates and governance pack are in place, so the next evolution is richer postmortems and more vendor-specific rollout packs.
