# 学习进度

## Current

- current_topic: agent evaluation methodology across task taxonomy, trace grading, regression suites, online feedback, and open-source tooling for AgentOps
- current_goal: turn agent evaluation from scattered eval concepts into a real engineering workbench with method, metrics, tooling, and repeatable release judgment

## What Just Got Stronger

- `Agent Runtime` 这条线现在已经不只讲 architecture，还补上了 failure cases、postmortems、rollout gates 和 blast radius control
- 新增 `Plugin、Memory 与 Background Task 失效模式`，把最容易隐藏问题的三层单独拆出来讲
- 现在又把 memory 正式拆成一条子主线：`记忆架构：State、Memory、Artifact 与 Context`、`记忆写入、召回、压缩与 Consolidation`、`记忆评测、污染、遗忘与纠偏`
- `Memory Lab` 也补上了分层、写入策略、准自进化实验，以及 learnings -> promotion -> skill extraction 实验
- 现在还把最小 `self-improving-learning-ledger` skill 脚手架真正落地了：可记录 learning、生成 promotion report、产出 review-only skill stub
- 这一轮继续把它推进到了 `incident-fed eval`、`shadow rollout`，并在本地做了最小 plugin wrapper scaffold
- 现在又把它推进到了更像正式门禁的一层：consolidation、manual verify、cluster-based candidate skill、incident pack replay、expectation checks
- 这一轮又把 prompt / context / tools / CLI / skills / plugins / harness 的工程分层单独讲清楚了，方便后面继续拆细子域
- 现在又把它继续拆成了 `Prompt 到 Context`、`动作面设计`、`扩展面设计` 三个更适合工程学习和教学的子域页
- 这一轮又把全局 `当前工作台` 和 `Self-Improving Memory Lab 验收与当前边界` 补出来，跨线程恢复和插件验收边界都清楚了
- 这一轮又补出了一条更完整的 `Agent Evaluation` 子主线：新增 `Agent 效果评估：方法论、开源工具与判断框架`，把任务分层、trace grading、online feedback、release gate 和开源工具组合真正收成了工作台
- 同时补上了 `Ragas`、`Inspect AI`、`Giskard` 三张系统卡，让 agent eval 不再只停留在 `Langfuse / Phoenix / Promptfoo`

## Why This Matters

- 现在 AI-Engineering 不再只是训练深、harness 深，而 runtime 下半场明显补齐了
- 这让 agent engineering 不只会谈 capability，还会谈失效、回滚、恢复和 rollout discipline
- 后面继续补 incident library 时，会更像持续积累的运行经验库，而不是新的概念页
- 这一轮已经把 self-improving memory 的 risk/governance 层补到位：incident library、shared-memory boundary、eval/release gate、rollback
- 更重要的是，`Memory Lab` 现在不只是“懂理论”，而是有了可回放的本地 plugin lab
- 现在 `AI-Engineering` 的评测层也更完整了：不只是通用 eval，而是明确有了 agent 任务评估、工具选型和效果判断框架
