# AI Interview Kit 学习进度

## Current

- current_topic: interview knowledge base with coverage-audit layer
- current_goal: make the module not only usable for preparation, but also auditable, so we can explicitly judge where coverage is strong and where it is still thin

## Completed

- Built the first full interview submodule with `专题总览 / 学习进度 / 恢复笔记`.
- Added a true interview opening layer with `开场与项目深挖题库`, so the pack now starts from self-introduction, project deep dive, owner boundary, failure handling, and senior-level positioning.
- Added a `团队与业务沟通题库`, so the pack now covers management-facing and cross-functional questions instead of stopping at pure technical depth.
- Added `资深候选人表达总纲`, so the module now teaches how to sound like a senior owner before drilling into individual topics.
- Added an expert-style detailed answer playbook so preparation can move from answer skeletons to complete interview-ready responses.
- Added follow-up drill packs: interviewer follow-up scripts, candidate counter-question templates, and common wrong-answer corrections.
- Added `面试问题导航` and `面试决策导航` so preparation can start from either technical question or architecture decision.
- Added a full knowledge-base layer with `完整面试题蓝图` plus three missing high-leverage banks: `系统设计与场景`、`成本、性能与容量规划`、`项目复盘与事故`.
- Added company-style scenario packs for `企业知识库 / AI Coding Agent / 客服 Copilot`, so the knowledge base now supports realistic company-context interview rehearsal instead of only generic question coverage.
- Added `题目覆盖审查：按五条主干与面试维度`, so the module can now judge its own coverage instead of relying on subjective confidence.

## Next

- Thicken the missing long-tail lanes first: `multimodal / voice / OCR / computer use` and deeper `rollout / ROI / governance` questions.
- Add one more layer of scenario packs later for `voice/realtime`, `document AI`, and possibly `browser/computer-use`, if those become target interview lanes.
