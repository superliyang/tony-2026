# 面试决策导航

## 决策 0：自我介绍先讲技术栈，还是先讲结果？

- 适用场景：开场 1-2 分钟自我介绍。
- 判断标准：
  - 高级岗位优先讲 `主线 + 代表结果 + 你的独特价值`，技术栈放到后面点到为止。
  - 只有在岗位非常偏底层研究时，才把技术方向放到更前面。
- 关联题库：[[./05-Question-Banks/00-开场与项目深挖题库|开场与项目深挖题库]]

## 决策 1：是继续堆 Prompt，还是升级到 Harness/Eval 体系？

- 适用场景：回答质量波动大、上线后频繁回归。
- 判断标准：
  - 若失败模式可枚举且可回归，优先升级 Eval/Harness。
  - 若需求还在探索，先保留轻量 Prompt 迭代。
- 关联题库：[[./05-Question-Banks/05-评测与上线治理题库|评测与上线治理题库]]

## 决策 2：长上下文 vs RAG，优先做哪个？

- 适用场景：多轮对话、文档问答质量下降。
- 判断标准：
  - 若核心问题是“记住历史上下文”，优先长上下文策略。
  - 若核心问题是“知识更新快且体量大”，优先 RAG + 改写 + rerank。
- 关联题库：[[./05-Question-Banks/02-推理与系统架构题库|推理与系统架构题库]]、[[./05-Question-Banks/04-RAG与数据工程题库|RAG与数据工程题库]]

## 决策 3：单 Agent 还是多 Agent？

- 适用场景：任务复杂、步骤依赖多、工具类型多。
- 判断标准：
  - 若任务可线性执行，优先单 Agent 降复杂度。
  - 若有明显并行子任务和专业化收益，再上多 Agent。
- 关联题库：[[./05-Question-Banks/03-Agent与工具调用题库|Agent与工具调用题库]]

## 决策 4：安全优先导致体验下降，怎么权衡？

- 适用场景：高风险业务（金融、医疗、客服自动外发）。
- 判断标准：
  - 高风险动作强制 HITL 与审批门禁。
  - 低风险动作采用自动化策略与抽样复核。
- 关联题库：[[./05-Question-Banks/06-安全与合规题库|安全与合规题库]]

## 决策 5：面试里先讲成功结果，还是先讲风险与事故？

- 适用场景：项目深挖、上线治理、资深岗位追问。
- 判断标准：
  - 先用 1 句话交代结果，证明项目不是失败样本。
  - 立刻补上风险、事故和治理动作，证明你不是只会讲喜报。
- 关联题库：[[./05-Question-Banks/00-开场与项目深挖题库|开场与项目深挖题库]]、[[./05-Question-Banks/07-团队与业务沟通题库|团队与业务沟通题库]]
