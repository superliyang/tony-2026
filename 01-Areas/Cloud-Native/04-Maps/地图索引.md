## 地图怎么用

- `云原生学习路径图`：回答先学什么、后学什么
- `云原生技术栈图`：回答各主题和项目在整条栈上的位置
- `云原生平台与治理图`：回答 Kubernetes 之后，平台、流量和安全能力是如何长出来的
- `云原生平台实践图`：回答平台工程如何从概念变成 portal、catalog、golden path 和团队 workflow
- `平台采用与度量图`：回答平台产品化以后，adoption、metrics、治理和 roadmap 如何闭环
- `云原生运行时与状态图`：回答 Kubernetes 之外，runtime、serverless 和 storage 这三条基础设施扩展线如何展开
- `AI 时代云原生栈图`：回答 AI workload 如何在 Kubernetes 之上长出推理、训练和加速器编排层
- `AI 时代云原生案例图`：回答这些 AI-native 平台能力在案例层如何分化成推理、训练和 distributed execution 三条路径
- `服务网格与安全落地图`：回答 service mesh 和 security 是怎样从概念层进入 workflow 和 case 层的
- `运行时与状态案例图`：回答 runtime、serverless 和 storage 在案例层分别体现成什么平台形态
- `AI 时代云原生采用图`：回答组织是如何从实验阶段一路走到 AI platform operating model 的
