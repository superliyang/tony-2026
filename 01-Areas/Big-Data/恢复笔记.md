## What I Actually Understand Now

- 大数据不是单一技术，而是围绕数据生命周期展开的系统工程
- 学习顺序应该先抓：数据生命周期 -> 数据建模 -> batch / streaming -> lakehouse / warehouse -> 平台治理
- 大数据和 Cloud-Native、AI-Engineering 有强连接，但边界不同：这里关注数据怎么可靠地进入决策和系统动作
- batch / streaming 的核心不是新旧，而是延迟、状态、正确性、回放和成本 trade off
- warehouse / lake / lakehouse 的核心不是代际替换，而是分析体验、开放存储、多引擎复用和治理能力的取舍

## What Still Feels Fuzzy

- Spark / Flink / Kafka / Iceberg / Airflow / dbt / Trino 等系统还没有逐个建立卡片
- 实时链路、数仓分层、lakehouse 表格式和数据治理还需要后续拆专题
- 真实业务案例还没铺开，后续最好从支付、游戏、AI 数据平台中选一个做贯穿案例
